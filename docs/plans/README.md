ARX Website Planning Workspace

Purpose
- Keep redesign strategy, phase checklists, and design rationale in one place.
- Make iterative visual work safe for production by shipping in controlled phases.

Structure
- docs/plans/README.md (this file)
- docs/plans/2026-04-12-retro-minecraft-redesign-plan.md (master implementation plan)
- docs/plans/phases/phase-1-campfire-ops.md (active phase execution checklist)

Execution Rules
1) Build and ship one phase at a time.
2) Run lint/build after each phase.
3) Keep docs readability and performance as non-negotiable constraints.
4) Any strong visual effects must degrade gracefully on mobile and reduced-motion.
